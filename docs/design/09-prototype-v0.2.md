# 09. 第一版极简原型开发说明（v0.2）

## 1) 原型目标

本版本仅验证以下四项：

1. 主炮瞄准与射击手感
2. 天体碎裂与碎块掉落反馈
3. 部分碎块捕获并落地转资源的闭环
4. 中文 UI 清晰度与波后收益/损失可读性

本版本不验证：复杂科技树、制导、激光、火箭回收、复杂波次差异。

## 2) 本版本范围

### 2.1 必做

- 1 张地图
- 1 个主炮塔
- 1 种武器：无制导火炮
- 1 套天体生成方式
- 1 套通用碎裂系统
- 1 套碎块捕获/掉落/转资源逻辑
- 1 套中文 UI
- 1 个波次系统基础壳
- 1 个极简波后结算

### 2.2 暂不实现

- 激光
- 制导
- 火箭回收
- 火箭发射台
- 多地图差异
- 复杂目标速度测试
- 复杂目标规模测试
- 小行星类型细分玩法差异
- 科技树多分支
- 生态建设
- 数值平衡细调

## 3) 平台与技术要求

| 项目 | 说明 |
| --- | --- |
| 平台 | 浏览器 PC |
| 渲染 | HTML5 Canvas |
| UI | 中文优先，HTML/CSS 或 Canvas 均可 |
| 美术 | 纯程序生成 |
| 输入 | 鼠标 |
| 多语言 | 预留中英切换按钮，英文可先空壳 |

## 4) 最小玩法闭环（单波）

1. 显示“下一波开始”
2. 天体进入屏幕
3. 玩家用无制导火炮瞄准射击
4. 天体受击后按参数碎裂
5. 部分碎块进入捕获区并下落
6. 落地后转化为资源
7. 未捕获或烧蚀碎块计入损失
8. 波后显示资源收益与资源损失
9. 进入下一波

## 5) 地图设计

### 地图 A：稀薄大气测试场（仅此一张）

表现：
- 屏幕下方约 40%~50% 星球弧面，上方为空域
- 轻微大气视觉层
- 捕获逻辑偏宽松

目的：
- 降低环境干扰
- 优先验证“打碎 -> 捕获 -> 落地 -> 收益/损失”

## 6) 核心设施

### 6.1 主炮塔

功能：
- 唯一可操作设施
- 可旋转瞄准
- 发射无制导炮弹
- 命中触发碎裂

操作：
- 鼠标移动：瞄准
- 鼠标左键点击：发射
- 可选：按住连续发射（实现简单可保留）

本版不做：模式切换、波间配置、高级武器切换。

## 7) 天体设计（简化版）

统一生成原则：
- 全部天体使用同一结构：多个大多边形主体 + 若干小多边形附着块
- 不做复杂独立模板系统
- 差异仅通过参数与表现控制

组成：
- 大多边形主体：2~5 个，承担主要质量与轮廓
- 小多边形附着块：若干，用于轮廓细节与小碎裂

复用原则：
- 统一碎裂框架，不为每类目标单写算法
- 参数化项：主体块数、附着块数、总体尺寸、连接紧密度、易碎度、颜色/表面表现

## 8) 碎裂系统（核心）

目标：
- 统一、可复用、可参数驱动
- 当前仅接无制导火炮
- 依据打击参数与天体参数输出碎裂结果

输入参数：

| 参数 | 来源 | 含义 |
| --- | --- | --- |
| impact_power | 打击参数 | 命中强度 |
| impact_point | 打击参数 | 命中位置 |
| impact_direction | 打击参数 | 命中方向 |
| body_size | 天体参数 | 天体整体尺寸 |
| body_fragility | 天体参数 | 易碎程度 |
| body_cohesion | 天体参数 | 连接强度 |
| polygon_layout | 天体参数 | 当前拼接结构 |

输出结果：
- 大碎块：数量少、价值高、捕获率更高
- 小碎块：数量多、价值低、更易飞散/损失

第一版表现：
- 命中点附近优先崩裂
- 生成散开的大/小碎块
- 偏爆裂式，不做精确切割

扩展接口预留：
- `damage_mode = explosive`（v0.2 启用）
- `damage_mode = cutting`（预留）
- `damage_mode = guided_penetration`（预留）

## 9) 碎块行为

每个碎块至少具备：

| 属性 | 说明 |
| --- | --- |
| position | 位置 |
| velocity | 速度 |
| size | 尺寸 |
| mass_value | 资源价值 |
| captured | 是否已进入捕获状态 |
| lost_reason | 若损失，记录原因 |
| landed | 是否已落地 |

## 10) 捕获与损失逻辑

### 10.1 捕获

- 在星球上方定义抽象“捕获区”
- 碎块进入后逐步转为下落态
- 落地后转资源

原则：
- 大碎块捕获概率 > 小碎块
- 低速碎块更易捕获
- 第一版整体宽容

### 10.2 损失

| 损失类型 | 含义 |
| --- | --- |
| 未捕获 | 未进入捕获区并飞离屏幕 |
| 烧蚀掉 | 下落过程中损失部分或全部 |

简化建议：
- 小碎块更易未捕获
- 极小碎块或高速小碎块更易烧蚀

### 10.3 资源结算

- 落地成功：计入资源收益
- 未成功：计入资源损失

## 11) 波次系统（极简）

目标：保留波次节奏，不测复杂变量。

做法：
- 每波固定数量天体
- 波间短暂停顿
- 难度轻微递增

参数接口先留在数据结构：

```js
speed
size
fragility
value
spawnCount
```

## 12) 科技树（极简壳）

目标：验证“波后拿资源 -> 点一次科技 -> 火炮反馈增强”。

节点不超过 3 个：

| 节点 | 效果 |
| --- | --- |
| 金属炮管 | 提高射速或稳定性 |
| 无烟火药 | 提高炮弹速度或命中体验 |
| 高装药火炮 | 提高碎裂强度 |

原则：不做复杂前置图，仅纵向小列表。

## 13) UI 要求

### 13.1 语言

- 中文优先
- 保留“中文 / English”切换按钮
- 英文可先空壳或少量字段

### 13.2 主界面至少包含

| 区域 | 内容 |
| --- | --- |
| 顶部 | 当前波次、开始/暂停 |
| 左上或右上 | 当前资源 |
| 中央战场 | 星球、炮塔、天体、碎块 |
| 底部或侧边 | 科技按钮 / 语言切换按钮 |

### 13.3 波后结算（仅两项）

| 项目 | 说明 |
| --- | --- |
| 资源收益 | 成功回收到的资源 |
| 资源损失 | 未捕获 / 烧蚀掉的资源 |

文案示例：
- 本波资源收益：`+23`
- 本波资源损失：`12`
- 附加小字：未捕获 `8`，烧蚀 `4`

## 14) 低美工表现规范

原则：信息清楚优先、程序生成优先、不做正式打磨。

| 对象 | 表现 |
| --- | --- |
| 星球 | 单个大圆弧 + 两层颜色 |
| 炮塔 | 简单底座 + 旋转炮管 |
| 天体 | 多个大多边形 + 小多边形拼接 |
| 碎块 | 小多边形 |
| 炮弹 | 小亮点或短线 |
| 爆裂 | 简单闪光 + 速度散开 |
| 落地资源 | 小闪点 + 数字跳字 |

## 15) 推荐开发顺序

### Phase 1：基础骨架

1. Canvas 主循环
2. 星球背景
3. 主炮塔瞄准与开火
4. 单个天体生成与飞行

### Phase 2：核心机制

5. 多边形拼接天体
6. 命中判定
7. 通用碎裂系统
8. 大碎块/小碎块生成
9. 捕获区与下落
10. 落地转资源
11. 未捕获/烧蚀损失统计

### Phase 3：外层流程

12. 波次流程
13. 波后结算
14. 小型火炮科技树
15. 中文 UI
16. 中英切换按钮壳子

## 16) Done Definition（v0.2）

满足以下条件即视为完成：

- 至少可连续进行 5 波
- 可鼠标瞄准无制导火炮并射击
- 天体由“大多边形主体 + 小多边形附着块”构成
- 命中后可基于参数碎裂
- 可生成大碎块和小碎块
- 部分碎块可进入捕获区并落地
- 落地可转化为资源
- 未捕获和烧蚀损失可统计
- 波后可显示“资源收益 / 资源损失”
- UI 默认中文且存在中英切换按钮
