# 00. 开发发布与分支规范（最高优先级）

> 本文档为跨版本强制规范，优先级高于一般实现细节。

## 1) 分文件开发 + 单 HTML 分发（强制）

- 开发过程必须采用分文件模块化结构。
- 每次迭代都必须打包一个可直接运行的单 HTML 分发版本。
- 单 HTML 分发包应与源码版本一一对应，便于回溯。

## 2) 版本号规范（强制）

- 分发版本号采用三段语义化格式：`MAJOR.MINOR.PATCH`（例如 `1.0.0`）。
- 在没有特别说明时，版本号连续递增管理（通常递增 `PATCH` 或 `MINOR`）。
- 当发生重大改动时，由用户明确通知手动提升大版本（例如 `1.1.0`、`2.0.0`）。

## 3) 重大改动分支策略（强制）

- 重大改动必须使用 Git 新分支开发。
- 保持之前稳定分支可用，不在其上直接破坏性修改。
- 重大改动完成并稳定后，将稳定成果合并回主分支。

## 4) 建议执行清单（每次发版）

1. 确认本次版本号（`x.y.z`）。
2. 完成分文件开发与自测。
3. 打包单 HTML 分发版本并标注版本号。
4. 记录本次变更到 `99-change-log.md`。
5. 若为重大改动：确认在新分支开发并完成合并流程。
